# The Synthesis Ritual

**Purpose**: Assembling the final deliverable from multi-stage work. The Synthesis ritual takes the outputs of all castes and composes them into a coherent whole.

---

## When to Invoke

- CHAIN tier: assembling multiple stage prompts into a pipeline document
- Hive spawning: assembling all daughter hive files into a complete package
- Complex outputs with multiple components (prompt + validation + documentation)

## Synthesis Protocol

```
SYNTHESIS

Phase 1: INVENTORY
  What components exist?
  → Builder's output(s): prompts, chains, definitions
  → Guardian's verdict: validation evidence
  → Sentinel's report: systemic check results
  → Nurse's state design: handoff schemas (if chain)

Phase 2: ASSEMBLY
  Compose into the target output form:
  → Read the appropriate output-forms/ file
  → Place each component in its designated location
  → Verify no component references state/context that isn't available

Phase 3: COHERENCE CHECK
  The Connector role (performed by the Sentinel during Synthesis):
  → Do all components use consistent terminology?
  → Do cross-references resolve correctly?
  → Is the information hierarchy clear? (most important first)
  → Does the assembled whole tell a coherent story?

Phase 4: COMPRESSION
  The Optimizer pass (performed by the Builder during Synthesis):
  → Remove redundancy between components
  → Merge overlapping constraints
  → Compress examples that demonstrate the same point
  → Verify every token earns its place

Phase 5: DELIVERY
  Present the assembled output with:
  → The output itself (prompt, chain doc, hive blueprint)
  → Validation report (from soldiers)
  → Scope definition (in/out/boundary)
  → Iteration guidance (how to evolve this output)
```

## Chain Assembly Pattern

For CHAIN outputs, the Synthesis ritual assembles:

```
1. Architecture overview (flow diagram, stage count, data flow)
2. Each stage:
   - System prompt (from Builder)
   - User message template (from Builder)
   - Output schema (from Builder + Nurse)
   - Pass condition (from Guardian)
   - Failure behavior (from Guardian)
3. Orchestration notes:
   - State management (from Nurse)
   - Error propagation (from Guardian + Sentinel)
   - Human checkpoints (from Scout's terrain map)
   - Parallelizable stages (from Nurse)
4. Validation evidence (from all soldiers)
5. Iteration log (if this is an evolution)
```

## Hive Assembly Pattern

For spawning, the Synthesis ritual assembles:

```
1. SKILL.md (daughter queen)
2. Worker files (one per domain task)
3. Soldier files (at least one)
4. Drone files (if data-heavy domain)
5. Memory files (domain-specific patterns)
6. Colony registration entry (for queen/lineage.md)
```
