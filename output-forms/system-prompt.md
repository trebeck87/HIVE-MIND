# Output Form: System Prompt

The shape of a system prompt generated by the colony.

---

## Structure

```
You are [role with specific expertise boundaries].

<core_objective>
[Measurable goal with explicit scope — FIRST 50 tokens]
</core_objective>

<output_format>
[Exact structure with type annotations]
[If JSON: "Respond ONLY with valid JSON. No markdown, no backticks, no preamble."]
[If structured: template with field definitions]
</output_format>

<constraints>
- [Hard rule 1 — non-negotiable]
- [Hard rule 2 — non-negotiable]
[Max 6-8 constraints. Merge overlapping ones.]
</constraints>

<edge_case_handling>
- If [condition]: then [specific behavior]
- If [ambiguity]: then [resolution]
- If [contradiction]: then [default behavior]
</edge_case_handling>

<examples>
GOOD output:
[Complete example showing ideal behavior]
Why this works: [Annotation — what makes this good]

BAD output:
[Complete example showing common failure]
Why this fails: [Annotation — specific problems]
</examples>
```

## Required Elements (MEDIUM+)

| Element | Required At | Purpose |
|---|---|---|
| Role statement | All tiers | Identity and expertise scope |
| `<core_objective>` | All tiers | Front-loaded mission |
| `<output_format>` | All tiers | Structure specification |
| `<constraints>` | MEDIUM+ | Hard rules |
| `<edge_case_handling>` | COMPLEX+ | Conditional logic |
| `<examples>` (70% Rule) | MEDIUM+ | 2 good + 1 bad minimum |
| Validation evidence | MEDIUM+ | Proof it works (see validation-report.md) |

## Companion User Message Template

Every system prompt should be accompanied by its user message template:

```
USER MESSAGE TEMPLATE:

[Variable field 1]: {PLACEHOLDER}
[Variable field 2]: {PLACEHOLDER}
[Context that changes per call]: {PLACEHOLDER}

Example populated message:
[Show what a real call looks like with all fields filled]
```
